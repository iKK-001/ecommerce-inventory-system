# 办公室部署修正版

更新时间：2026-06-06

本文用于把当前电商库存系统部署到朋友办公室局域网内，让 4-5 台设备通过浏览器访问同一套库存、成本和每周销量数据。

## 1. 结论

推荐方案：一台办公室电脑作为服务器，其他电脑访问这台电脑的局域网地址。

建议优先使用 Docker 部署。如果现场电脑不方便安装 Docker，再使用本机 PHP 方式部署。

当前 GitHub 仓库不需要改成公开。保持私有即可，现场通过 GitHub 登录、协作者权限或只读 Token clone `main` 分支。

重要边界：

- GitHub 只同步代码，不同步数据库。
- 现在录入过的 SKU、成本、库存数据如果存在本机 SQLite 或 Docker volume 里，需要单独备份并带到现场。
- 服务器电脑关机、睡眠、断网或服务窗口关闭时，其他设备无法访问。
- 局域网 HTTP 访问必须设置 `SESSION_SECURE_COOKIE=false`，否则登录 Session 可能异常。

## 2. 现场准备清单

服务器电脑建议选择：

- Mac mini、台式机、固定放在办公室的 MacBook 或 Windows 主机。
- 上班期间保持开机、插电、不休眠。
- 优先接网线；没有网线再用 Wi-Fi。
- 硬盘至少预留 10GB。

提前准备：

- GitHub 仓库地址：`https://github.com/iKK-001/ecommerce-inventory-system.git`
- GitHub 私有仓库访问方式：你的账号登录、朋友账号 collaborator、或 fine-grained read-only token。
- 服务器电脑管理员权限。
- 朋友办公室 Wi-Fi / 路由器权限，最好可以固定服务器电脑 IP。
- 当前真实数据库备份。如果你本机已经录入 SKU 和成本，必须单独迁移数据库。

## 3. 推荐方案 A：Docker 局域网部署

适合现场希望少折腾 PHP、Composer、Node 版本的情况。

### 3.1 安装工具

Mac：

```bash
brew install git gh
```

安装 Docker Desktop：

```text
https://www.docker.com/products/docker-desktop/
```

安装后打开 Docker Desktop，确认状态为 running。

如果仓库仍是私有，先登录 GitHub：

```bash
gh auth login --web --scopes repo
```

### 3.2 clone main

```bash
mkdir -p ~/Projects
cd ~/Projects
git clone https://github.com/iKK-001/ecommerce-inventory-system.git inventory-system
cd inventory-system
git checkout main
git pull origin main
```

### 3.3 查看服务器局域网 IP

Wi-Fi 通常是：

```bash
ipconfig getifaddr en0
```

有线网络可能是：

```bash
ipconfig getifaddr en1
```

假设得到：

```text
192.168.1.88
```

后续员工访问地址就是：

```text
http://192.168.1.88:8000/weekly-sales
```

### 3.4 写入 Docker 局域网配置

在项目根目录创建或修改 `.env`，只用于 Docker Compose 端口：

```bash
printf "APP_PORT=8000\nMAILPIT_PORT=8025\n" > .env
```

创建 `docker-compose.override.yml`，把 `192.168.1.88` 换成现场服务器电脑真实 IP：

```bash
cat > docker-compose.override.yml <<'EOF'
services:
  app:
    environment:
      APP_URL: "http://192.168.1.88:8000"
      APP_ENV: "local"
      APP_DEBUG: "false"
      APP_LOCALE: "zh-CN"
      APP_FALLBACK_LOCALE: "en"
      SESSION_SECURE_COOKIE: "false"
  worker:
    environment:
      APP_URL: "http://192.168.1.88:8000"
      APP_ENV: "local"
      APP_DEBUG: "false"
      APP_LOCALE: "zh-CN"
      APP_FALLBACK_LOCALE: "en"
      SESSION_SECURE_COOKIE: "false"
  scheduler:
    environment:
      APP_URL: "http://192.168.1.88:8000"
      APP_ENV: "local"
      APP_DEBUG: "false"
      APP_LOCALE: "zh-CN"
      APP_FALLBACK_LOCALE: "en"
      SESSION_SECURE_COOKIE: "false"
EOF
```

说明：

- `.env` 和 `docker-compose.override.yml` 是现场机器本地配置，不需要提交到 GitHub。
- `APP_URL` 必须是其他员工电脑能访问的局域网地址。
- `SESSION_SECURE_COOKIE=false` 是局域网 HTTP 部署的关键配置。

### 3.5 启动系统

```bash
docker compose up -d --build
```

查看状态：

```bash
docker compose ps
```

查看日志：

```bash
docker compose logs -f app
```

第一次启动会自动：

- 构建前端资源。
- 生成 `APP_KEY`。
- 创建 SQLite 数据库文件。
- 执行数据库迁移。

### 3.6 首次初始化管理员

打开：

```text
http://192.168.1.88:8000/install
```

按页面提示创建组织和管理员账号。不要在真实办公室数据环境执行 `php artisan db:seed`，否则可能写入测试用户或 demo 数据。

创建完成后访问：

```text
http://192.168.1.88:8000/weekly-sales
```

## 4. 备用方案 B：Mac 本机 PHP 部署

适合现场不想用 Docker，或者 Docker Desktop 安装不方便的情况。

### 4.1 安装依赖

```bash
brew update
brew install git gh php@8.4 composer node
```

如果 `php -v` 不是 8.4，后续命令前加：

```bash
export PATH="/opt/homebrew/opt/php@8.4/bin:$PATH"
```

### 4.2 clone main

```bash
mkdir -p ~/Projects
cd ~/Projects
gh auth login --web --scopes repo
git clone https://github.com/iKK-001/ecommerce-inventory-system.git inventory-system
cd inventory-system
git checkout main
```

### 4.3 安装依赖和配置环境

```bash
composer install
npm install
cp .env.example .env
php artisan key:generate
touch database/database.sqlite
```

编辑 `.env`：

```bash
nano .env
```

建议配置如下，把 IP 和路径换成现场真实值：

```env
APP_NAME="库存管理系统"
APP_ENV=local
APP_DEBUG=false
APP_URL=http://192.168.1.88:8000
APP_LOCALE=zh-CN
APP_FALLBACK_LOCALE=en

DB_CONNECTION=sqlite
DB_DATABASE=/Users/<USERNAME>/Projects/inventory-system/database/database.sqlite

SESSION_DRIVER=database
SESSION_SECURE_COOKIE=false
SESSION_DOMAIN=null

CACHE_STORE=database
QUEUE_CONNECTION=database
```

初始化：

```bash
php artisan migrate --force
npm run build
```

### 4.4 启动服务

```bash
php artisan serve --host=0.0.0.0 --port=8000
```

如果要防止 Mac 睡眠导致服务中断，可以用：

```bash
caffeinate -dimsu php artisan serve --host=0.0.0.0 --port=8000
```

终端窗口不要关闭。

## 5. 员工电脑访问

员工电脑必须和服务器电脑在同一个办公室局域网内。

访问：

```text
http://服务器电脑局域网IP:8000/weekly-sales
```

示例：

```text
http://192.168.1.88:8000/weekly-sales
```

建议让路由器固定服务器电脑 IP，例如固定为 `192.168.1.88`。路由器后台常见名称：

- DHCP 静态分配
- 地址保留
- 固定 IP
- IP-MAC 绑定

固定后，员工每天都访问同一个地址。

## 6. 真实数据迁移

### 6.1 如果当前数据在本机 SQLite

先关闭本机 Laravel 服务，然后复制：

```text
database/database.sqlite
```

到现场服务器项目同一路径：

```text
/Users/<USERNAME>/Projects/inventory-system/database/database.sqlite
```

确认现场 `.env` 里的 `DB_DATABASE` 指向这个文件的完整路径。

### 6.2 如果当前数据在 Docker volume

在当前机器项目目录执行：

```bash
mkdir -p ~/Desktop/inventory-backups
docker compose cp app:/app/database/database.sqlite "$HOME/Desktop/inventory-backups/database_$(date +%Y-%m-%d_%H-%M-%S).sqlite"
```

把导出的 `.sqlite` 文件带到现场，再恢复到现场 Docker 容器：

```bash
docker compose cp /path/to/database.sqlite app:/app/database/database.sqlite
docker compose restart
```

### 6.3 重要提醒

不要以为 `git push` 或 `git clone` 会同步库存数据。GitHub main 只负责代码，真实业务数据必须通过数据库备份迁移。

## 7. 每天启动脚本

### 7.1 Docker 启动脚本

在桌面创建 `启动库存系统.command`：

```bash
#!/bin/bash
cd "$HOME/Projects/inventory-system"
docker compose up -d
IP="$(ipconfig getifaddr en0)"
open "http://${IP}:8000/weekly-sales"
```

授权：

```bash
chmod +x ~/Desktop/启动库存系统.command
```

### 7.2 本机 PHP 启动脚本

在桌面创建 `启动库存系统.command`：

```bash
#!/bin/bash
cd "$HOME/Projects/inventory-system"
export PATH="/opt/homebrew/opt/php@8.4/bin:$PATH"
caffeinate -dimsu php artisan serve --host=0.0.0.0 --port=8000
```

授权：

```bash
chmod +x ~/Desktop/启动库存系统.command
```

## 8. 备份方案

库存、销量、成本数据比代码更重要。建议每天至少下班前备份一次，并复制到 U 盘、另一台电脑或网盘。

### 8.1 Docker SQLite 备份

在项目目录执行：

```bash
mkdir -p ~/Desktop/inventory-backups
docker compose cp app:/app/database/database.sqlite "$HOME/Desktop/inventory-backups/database_$(date +%Y-%m-%d_%H-%M-%S).sqlite"
```

### 8.2 本机 SQLite 备份脚本

在桌面创建 `备份库存数据库.command`：

```bash
#!/bin/bash
PROJECT_DIR="$HOME/Projects/inventory-system"
BACKUP_DIR="$HOME/Desktop/inventory-backups"
mkdir -p "$BACKUP_DIR"
cp "$PROJECT_DIR/database/database.sqlite" "$BACKUP_DIR/database_$(date +%Y-%m-%d_%H-%M-%S).sqlite"
open "$BACKUP_DIR"
```

授权：

```bash
chmod +x ~/Desktop/备份库存数据库.command
```

## 9. 现场验收清单

服务器电脑：

- 打开 `http://127.0.0.1:8000/weekly-sales` 或 `http://localhost:8000/weekly-sales`。
- 打开 `http://服务器局域网IP:8000/weekly-sales`。
- 确认可以登录。
- 确认 SKU、售价、商品成本、国内物流、美国头程、美国尾程、打包成本、库存数据正常显示。
- 修改一个测试 SKU 成本并保存。
- 刷新页面后确认数据仍然存在。
- 录入一条测试销量，确认库存扣减逻辑正常。

员工电脑：

- 连接同一个 Wi-Fi 或局域网。
- 打开 `http://服务器局域网IP:8000/weekly-sales`。
- 确认可以登录。
- 保存一条测试数据。
- 服务器电脑刷新后能看到同一套数据。

备份：

- 执行一次备份脚本。
- 确认备份文件存在。
- 把备份文件复制到服务器电脑之外的位置。

## 10. 常见问题排查

### 10.1 员工电脑打不开

检查：

- 员工电脑和服务器电脑是否在同一个 Wi-Fi / 局域网。
- 地址是否写成了服务器局域网 IP，而不是 `127.0.0.1`。
- 端口是否正确：`8000`。
- Docker 或 `php artisan serve` 是否还在运行。
- Mac 防火墙是否拦截 Docker、Terminal 或 PHP。
- 路由器是否开启访客网络隔离。如果员工连的是 Guest Wi-Fi，可能无法访问内网设备。

### 10.2 登录后又跳回登录页

优先检查：

```env
SESSION_SECURE_COOKIE=false
APP_URL=http://服务器局域网IP:8000
SESSION_DOMAIN=null
```

修改后重启服务。

Docker：

```bash
docker compose restart
```

本机 PHP：

```bash
php artisan config:clear
php artisan serve --host=0.0.0.0 --port=8000
```

### 10.3 页面样式不对或空白

本机 PHP 部署执行：

```bash
npm run build
php artisan view:clear
php artisan config:clear
```

Docker 部署执行：

```bash
docker compose up -d --build
```

### 10.4 数据突然不见

先确认是不是连到了另一套数据库：

- Docker 部署看 Docker volume。
- 本机部署看 `.env` 的 `DB_DATABASE`。
- 不要随便执行 `docker compose down -v`，这个命令会删除 Docker volume 数据。
- 不要随便删除 `database/database.sqlite`。

## 11. 权限建议

管理员：

- 负责商品、成本、库存、用户管理。

普通录入人员：

- 只负责每周销量录入。
- 不建议开放成本编辑权限。

成本维护人员：

- 可以修改售价、商品成本、国内物流、美国头程、美国尾程、打包成本。

当前 MVP 先保证老板和少数录入人员可用。等使用稳定后，再细化权限和页面入口。

## 12. 最短现场流程

1. 选一台固定电脑当服务器。
2. 安装 Docker Desktop、Git、GitHub CLI。
3. `gh auth login --web --scopes repo`。
4. clone 私有仓库 main。
5. 查服务器局域网 IP。
6. 写 `docker-compose.override.yml`，设置 `APP_URL` 和 `SESSION_SECURE_COOKIE=false`。
7. `docker compose up -d --build`。
8. 打开 `/install` 创建管理员。
9. 员工电脑访问 `http://服务器IP:8000/weekly-sales`。
10. 路由器固定服务器 IP。
11. 做一次真实数据迁移。
12. 做一次备份并确认备份文件可用。
